### ✅ Recovery & Stabilization Plan (For Your Existing Node/React App)

#### 🔧 Step 1: **Purge Vite’s Cache & Build Artifacts**
These stale chunks live in Vite’s internal cache or `dist/`.

```bash
# Stop dev server
npm run dev -- --stop  # or just Ctrl+C

# Clear Vite cache (critical!)
rm -rf node_modules/.vite
rm -rf .vite

# Clear dist if exists
rm -rf dist

# Reinstall problematic deps cleanly
npm uninstall html5-qrcode
npm install html5-qrcode@latest
```

> 💡 **Why?** Vite caches dynamic imports. If a dependency updates its internal chunk structure but the cache isn’t cleared, Vite serves broken references.

---

#### 🔧 Step 2: **Lock Down Dynamic Imports**
In your QR scanner component (`src/components/QrScanner.tsx` or similar), **avoid lazy/dynamic imports** that trigger Vite’s code-splitting unpredictably:

```tsx
// ❌ BAD: Triggers dynamic chunking
const Html5Qrcode = await import('html5-qrcode');

// ✅ GOOD: Static import (forces bundling into main chunk)
import { Html5Qrcode } from 'html5-qrcode';
```

If you must lazy-load for performance, use **explicit Vite preloading**:
```ts
// In main.tsx or customer.tsx
import('html5-qrcode').then(() => {
  // Safe to use now
});
```

---

#### 🔧 Step 3: **Disable Vite’s Auto-Optimize During Dev (Temporary)**
Add to `vite.config.ts`:
```ts
export default defineConfig({
  optimizeDeps: {
    exclude: ['html5-qrcode'], // Prevent Vite from optimizing this dep
  },
  build: {
    commonjsOptions: {
      include: [/html5-qrcode/, /node_modules/], // Force CJS handling
    }
  }
});
```

This prevents Vite from rewriting the dependency’s internal imports.
